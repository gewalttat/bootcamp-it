📋 Обзор проекта

Что такое WB Junior Clone?

  • Клон маркетплейса Wildberries - упрощенная версия для обучения
  • E-Commerce приложение - полный цикл: каталог → корзина → оформление
  • Демонстрация модульной архитектуры - логично организованный código
  • Двухуровневая архитектура - отдельно frontend и backend

Основные возможности:

  • Просмотр каталога товаров
  • Поиск и фильтрация товаров
  • Добавление товаров в корзину
  • Управление товарами (добавление, удаление) через админ-панель
  • Сохранение данных в localStorage и на сервере

Слои приложения:

  • Presentation Layer (UI) - HTML/CSS, отображение магазина
  • Business Layer - JavaScript модули (логика фильтрации, корзины)
  • Data Layer - API сервер, работа с БД

Паттерн MVC-подобный:

  • Model - структура товаров (api.js)
  • View - отображение (catalog.js, header.js)
  • Controller - логика обработки (filter.js, cart.js)

📁 Модульная архитектура Frontend

JavaScript разделен на 8 независимых модулей, каждый отвечает за свою задачу:

🌐 api.js

Загрузка данных с сервера, HTTP запросы

💾 storage.js

Работа с localStorage (сохранение корзины)

🛒 cart.js

Логика управления корзиной

📺 catalog.js

Отрисовка товаров на странице

🔍 filter.js

Фильтрация, сортировка товаров

📋 header.js

Функции хедера, поиск, корзина

👨‍💼 admin.js

Добавление/удаление товаров на сервере

🚀 main.js

Инициализация приложения, точка входа

Преимущества: код легко масштабировать, тестировать и поддерживать

🖥️ Backend архитектура

Express.js сервер (server.js)

const express = require('express'); const cors = require('cors'); const PORT = 3000; const app = express(); app.use(cors()); // Разрешаем запросы с фронтенда

REST API endpoints:

Метод Endpoint Описание
GET /api/products Получить все товары
POST /api/products Добавить новый товар
PUT /api/products/:id Обновить товар
DELETE /api/products/:id Удалить товар

База данных: db.json

Простой JSON файл с массивом товаров (вместо SQL БД) - удобно для обучения!

🔄 Поток данных в приложении

1️⃣ Загрузка товаров:

main.js → api.js → fetch('/api/products')
↓
server.js (Express) → readDB() из db.json
↓
catalog.js → renderGoods() → HTML в браузер

2️⃣ Добавление товара в корзину:

User clicks "Добавить в корзину"
↓
cart.js addToCart(id)
↓
storage.js saveCart() → localStorage
↓
UI update (счетчик товаров)

3️⃣ Создание нового товара через админ:

admin.js (форма) → api.js (POST)
↓
server.js → writeDB() в db.json
↓
response → loadGoods() → новый товар в каталоге

🎯 Ключевые концепции программирования

1. Модульность (Modularity)

  • Код разбит на отдельные файлы-модули
  • Каждый модуль отвечает за одну задачу (Single Responsibility)
  • Легко тестировать и переиспользовать

2. REST API и HTTP

  • GET - получение данных
  • POST - создание
  • PUT - обновление
  • DELETE - удаление

3. Асинхронность (Async/Await)

  • Fetch API для HTTP запросов
  • Промисы - обещания результатов
  • Неблокирующие операции

4. Локальное хранилище (localStorage)

  • Сохранение данных на клиенте (браузер)
  • Данные сохраняются после закрытия вкладки
  • Идеально для кэша и настроек

📚 Full Tech Stack

Frontend Stack:

  • HTML5 (семантическая разметка)
  • CSS3 (Grid, Flexbox, медиазапросы)
  • Vanilla JavaScript ES6+ (без фреймворков!)
  • Fetch API для HTTP запросов
  • localStorage для персистентности

Backend Stack:

  • Node.js (JavaScript runtime)
  • Express.js (веб фреймворк)
  • CORS middleware (разрешение кроссдоменных запросов)
  • fs модуль (работа с файлами)
  • JSON для хранения данных

Инструменты и Утилиты:

  • npm (менеджер пакетов)
  • package.json (конфигурация проекта)
  • Git (версионирование кода)

▶️ Запуск проекта

1. Установка зависимостей сервера:

cd server npm install

2. Запуск сервера:

npm start // Сервер запущен на http://localhost:3000

3. Открыть клиент:

cd ../client // Открыть index.html в браузере (или использовать Live Server)
