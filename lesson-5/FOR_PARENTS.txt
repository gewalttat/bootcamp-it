ЧТО ЭТО ВООБЩЕ?
════════════════════════════════════════════════════════════════════════════════

WB Junior Clone это интернет-магазин. Типа как Wildberries, Ozon или AliExpress. Можно 
смотреть товары, искать их, добавлять в корзину, и еще админ может добавлять новые товары.

Это не РЕАЛЬНЫЙ магазин - это учебный проект. Студент сам все запрограммировал, чтобы научиться 
как работают такие приложения.

Представьте что ваш ребёнок построил с нуля целый интернет-магазин. Со всеми функциями.
Это как если бы строитель построил целый дом и вам показывает как он работает.


КАКИЕ ЧАСТИ ЕСТЬ В ЭТОМ ПРОЕКТЕ?
════════════════════════════════════════════════════════════════════════════════

1. ВЕБ-СТРАНИЦА (что видит пользователь в браузере)
   └─ Это как визитка магазина, красивая и интерактивная
   └─ Когда вы нажимаете кнопку "Добавить в корзину" - что-то происходит тут

2. "МОЗГИ" ПРИЛОЖЕНИЯ (JavaScript код на компьютере)
   └─ Это как память и логика магазина
   └─ Когда вы кликаете - код "думает" и решает что делать
   └─ Помнит что у вас в корзине даже если вы закроете браузер

3. СЕРВЕР (удаленный компьютер в интернете)
   └─ Это как база данных магазина
   └─ Хранит информацию о всех товарах
   └─ Когда вы запрашиваете товары - сервер их отправляет

4. БА́ЗА ДАННЫХ (где хранятся товары)
   └─ Это просто список всех товаров в текстовом файле
   └─ Название, цена, картинка каждого товара


В РЕАЛЬНЫХ МАГАЗИНАХ КАК?
════════════════════════════════════════════════════════════════════════════════

Wildberries на самом деле:
- Тысячи серверов вместо одного
- Миллионы товаров в специальной БД вместо текстового файла
- Сложная логика вместо простой
- Разработано 100+ программистами вместо одного студента

Но СТРУКТУРА У ВСЕХ ОДИНАКОВАЯ:
Веб-страница ↔ Сервер ↔ База данных

Ваш ребёнок сделал упрощенную версию, но это именно та же идея!


КАК ЭТО РАБОТАЕТ НА ПРАКТИКЕ?
════════════════════════════════════════════════════════════════════════════════

Сценарий 1: Вы открываете магазин

1. Вы пишете в браузере: localhost:3000
2. Браузер запрашивает у сервера: "Покажи мне товары"
3. Сервер читает список товаров из файла
4. Сервер отправляет товары обратно в браузер (в формате JSON - просто список)
5. На экране появляются товары красиво оформленные


Сценарий 2: Вы добавляете товар в корзину

1. Вы видите iPhone, нажимаете "Добавить в корзину"
2. JavaScript код запоминает что вы выбрали этот товар
3. Корзина сохраняется в памяти браузера
4. Вы видите что в иконке корзины число изменилось (1 товар)
5. Если вы закроете браузер и откроете снова - корзина все ещё там!


Сценарий 3: Админ добавляет новый товар

1. Админ нажимает кнопку "➕ Добавить товар"
2. Появляется форма (название, цена, картинка)
3. Админ заполняет форму
4. Нажимает "Сохранить"
5. JavaScript отправляет данные на сервер
6. Сервер добавляет новый товар в список
7. Все пользователи видят новый товар когда открывают магазин


ЧТО ТАКОЕ "ЯЗЫКИ ПРОГРАММИРОВАНИЯ"?
════════════════════════════════════════════════════════════════════════════════

Программирование = писать инструкции для компьютера

Языки программирования = разные способы писать эти инструкции

В этом проекте используются:

HTML - язык для создания структуры страницы
└─ Это как кирпичи для дома
└─ Определяет где что будет (заголовок, кнопка, картинка)

CSS - язык для красивого оформления
└─ Это как окраска и дизайн дома
└─ Определяет цвета, размеры, расположение элементов
└─ Делает страницу красивой

JavaScript - язык для логики и интерактивности
└─ Это как "мозги" дома
└─ Когда вы кликаете на кнопку - JavaScript это обрабатывает
└─ Работает и в браузере И на сервере (Node.js)

В общем: HTML = структура, CSS = красивость, JavaScript = движение и логика

Express.js - это библиотека для JavaScript которая упрощает создание сервера

JSON - это просто способ записывать данные (список товаров, например)
└─ Как записать в тетрадь: { имя: "iPhone", цена: 95000 }

ВАЖНЫЕ КОНЦЕПЦИИ
════════════════════════════════════════════════════════════════════════════════

API (Application Programming Interface)
└─ Это договор между браузером и сервером
└─ Браузер говорит: "Дай мне товары"
└─ Сервер говорит: "Вот товары в формате JSON"
└─ Это как язык на котором они общаются

REST (в основе этого механизм)
└─ GET = получить информацию (безопасно, т.е. ничего не портит)
└─ POST = создать новую информацию
└─ PUT = обновить информацию
└─ DELETE = удалить информацию
└─ Это стандартный способ организации сервера

Модульность
└─ Код разбит на маленькие куски каждый отвечает за свое
└─ Например, один файл отвечает за корзину, другой за поиск
└─ Это делает код понятнее и проще исправлять

localStorage
└─ Это память браузера на вашем компьютере
└─ Магазин может сохранять там что угодно (например, корзину)
└─ Примерно как шоколад в кармане - он сохраняется пока вы ходите


КАК ЗАПУСТИТЬ?
════════════════════════════════════════════════════════════════════════════════

Очень просто:

1. Открыть терминал (черное окно где печатают команды)

2. Запустить сервер (то что на серверных умопомрачениях):
   cd server
   npm install  (установить зависимости, типа как скачать программы которые нужны)
   npm start    (запустить сервер)

3. Открыть браузер и пойти на http://localhost:3000

4. Готово! Вы видите магазин!

Сервер работает на вашем компьютере (не в интернете). localhost = "мой компьютер".
